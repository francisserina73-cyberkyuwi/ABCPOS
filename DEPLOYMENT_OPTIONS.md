### **Option 2: VPS Deployment** (More Control)

1. **Setup Server**
   ```bash
   # Update system
   sudo apt update && sudo apt upgrade -y
   
   # Install LAMP stack
   sudo apt install apache2 mysql-server php php-mysql php-gd php-mbstring php-fileinfo -y
   
   # Enable Apache modules
   sudo a2enmod rewrite
   sudo systemctl restart apache2
   ```

2. **Configure MySQL**
   ```bash
   sudo mysql_secure_installation
   # Create database
   mysql -u root -p
   CREATE DATABASE pos_system1;
   CREATE USER 'pos_user'@'localhost' IDENTIFIED BY 'strong_password';
   GRANT ALL PRIVILEGES ON pos_system1.* TO 'pos_user'@'localhost';
   FLUSH PRIVILEGES;
   ```

3. **Upload Files**
   ```bash
   # Use SCP or SFTP
   scp -r pos_system_dark/* user@your-server:/var/www/html/
   
   # Or use Git
   git clone your-repo /var/www/html/
   ```

4. **Set Permissions**
   ```bash
   sudo chown -R www-data:www-data /var/www/html/
   sudo chmod -R 755 /var/www/html/
   sudo chmod -R 775 /var/www/html/uploads/
   sudo chmod -R 775 /var/www/html/logs/
   ```

5. **Configure Apache**
   ```bash
   # Create virtual host
   sudo nano /etc/apache2/sites-available/pos-system.conf
   ```

## ğŸ†˜ **Troubleshooting Common Issues**

### **Issue: Database Connection Error**
- Check database credentials sa `config.php`
- Verify database name, user, password
- Check if database server is running
- Check port (usually 3306, not 3307)

### **Issue: Permission Denied on Uploads**
- Set `uploads/` directory to 755 or 775
- Check owner (should be web server user)
- Some hosts need 777 (less secure)

### **Issue: 500 Internal Server Error**
- Check `logs/error.log`
- Check file permissions
- Check PHP error logs sa hosting panel
- Verify `.htaccess` is correct

### **Issue: Images Not Displaying**
- Check `uploads/images/` permissions
- Verify image paths are relative
- Check `.htaccess` in uploads folder
- Verify file ownership
